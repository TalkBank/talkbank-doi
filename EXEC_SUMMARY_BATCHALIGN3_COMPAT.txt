1. CANONICAL VS COMPAT OPTION REPRESENTATION & RESOLUTION
================================================================================

ARCHITECTURE: Three-tier resolution model

  Parse Layer (clap enums + hidden bool flags)
    ↓
  Resolution Layer (src/args/options.rs::build_typed_options)
    ↓
  Canonical CommandOptions (enum with string engine names)
    ↓
  Dispatch/Server (no compat awareness)

KEY RESOLUTION RULES:

A) Engine Names (Precedence: HIGHEST to LOWEST)
   1. Custom engine (--*-engine-custom <STR>)
   2. Compat bool flag (--whisper, --rev, etc.)
   3. Enum default (--asr-engine rev, default Rev)

B) Boolean Pairs (AND gate logic)
   final_value = positive_flag && !negative_flag
   
   Example: --wor && !--nowor (default true)
   Edge case: Both true → AND gate produces false

C) Special Case: Diarization (Transcribe only, two-stage check)
   Checks: --diarize → --nodiarize → enum
   If both compat flags: --diarize wins

HIDDEN COMPAT FLAGS SUMMARY (37 TOTAL):
  • Global: 10 flags (all no-ops: --memlog, --mem-guard, etc.)
  • Align: 4 flags (--whisper, --rev, --whisper-fa, --wav2vec)
  • Transcribe: 6 flags (--whisper, --whisperx, --whisper-oai, --rev, --diarize, --nodiarize)
  • Benchmark: 3 flags (--whisper, --whisper-oai, --rev)

KEY FILES:
  • Definition: src/args/global_opts.rs:15-60, commands.rs:81-394
  • Resolution: src/args/options.rs:1-278 (build_typed_options function)
  • Golden Test: tests/command_surface_manifest.rs (37 cases verified)

4. CAVEATS ABOUT HIDDEN FLAGS & HELP BEHAVIOR
================================================================================

CLAP HIDE MECHANISM:
  • #[arg(hide = true)] removes flag from --help output
  • Flag is FULLY PARSED and FULLY FUNCTIONAL
  • Not an error condition; intentional compatibility feature
  • Users must know flag name to use it (discovery via external docs)

BOOLEAN PAIR AND LOGIC:
  • final = positive && !negative (clap conflicts_with enforces mutual exclusion)
  • Both true → AND gate produces false (unintuitive but consistent)
  • Example: batchalign3 align --wor --nowor input/ → wor=false

GLOBAL COMPAT FLAGS ARE TRUE NO-OPS:
  • 10 flags parse but are never read in code
  • Kept for script compatibility only
  • No functional effect on behavior
  • Fields: memlog, mem_guard, adaptive_workers, pool, shared_models, etc.

DIARIZATION SPECIAL CASE (Transcribe only):
  • Two-stage check with explicit precedence
  • Checks --diarize BEFORE --nodiarize (not symmetric)
  • If both: --diarize wins (output variant TranscribeS vs Transcribe)
  • Asymmetric but intentional design

ENGINE NAME RESOLUTION ORDER:
  • Checked left-to-right through if/else chain
  • First non-None/false wins (not balanced precedence)
  • Custom > compat > enum always holds

HELP OUTPUT SCOPE:
  • Global compat flags hidden from top-level --help
  • Command-specific compat flags hidden from command-specific --help
  • Help scope can be either --help or subcommand --help
  • Test verifies both contexts

NO UNINTENDED HELP LEAKAGE:
  • Manifest test explicitly checks help output does NOT contain hidden flags
  • Both via grep string match and help_output() function call
  • Tests all 37 cases comprehensively

STABLE HIDDEN SURFACE GUARANTEE:
  • Golden test: tests/command_surface_manifest.rs:330-342
  • Runs on every CI build
  • Fails if new hidden flag added without test case
  • Enforces surface stability
