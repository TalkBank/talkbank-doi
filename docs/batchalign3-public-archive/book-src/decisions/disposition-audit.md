## Part A: Generated Data Files (git rm'd -- truly deleted)

These files are outputs of specific investigative workflows. The files themselves
are disposable, but some of the *workflows* that produced them are worth documenting.

### A1. Correctness Comparison Artifacts (`docs/bench_correctness_diffs/`)

**35 files:** 28 `.cha` outputs (14 align branch, 14 Python master), 6 `.diff` files,
`summary.md`, `metadata.json`, `benchmark.log`.

**What they were:** Side-by-side output from running both branches on the same test
datasets (align_small, morphotag_small, morphotag_medium), plus diffs showing
behavioral differences. Generated by the old `tools/run_full_benchmark.sh` script
(since removed).

**Purpose:** One-time validation that the Rust branch produced correct output
relative to Python master. The diffs documented expected differences (MWT
retokenization, timing nondeterminism) vs actual bugs.

**Still valid?** No. The workflow is superseded by `~/batchalign-benchmarking/`
(external repo with `run_benchmarks.py`, `compare_results.py`,
`cross_branch_benchmark.py`). The old in-repo scripts were already removed.
The specific diff findings (MWT retokenization, timing nondeterminism) are
documented in `python-rust-discrepancies.md`.

**Book action:** None needed. The cross-branch comparison workflow is already
documented in CLAUDE.md under "Benchmarking" and in the external benchmarking
repo's own README.

### A2. Corpus Audit File Lists

#### `FILES_NEED_ALIGN.txt` (3,510 paths)

**What it was:** List of CHAT files needing re-alignment due to broken `%wor`
tiers from the Python `decode()` bug.

**Purpose:** Operational checklist for the corpus remediation campaign.

**Still valid?** The workflow (scan corpus for %wor errors, produce a re-alignment
list) is a repeatable operational task. The specific list is spent (machine-specific
absolute paths). **If the remediation is complete, no action. If not, the list
can be regenerated.**

#### `FILES_NEED_MORPHOTAG.txt` (54,265 paths + 50-line header)

**What it was:** List of CHAT files needing re-morphotagging due to Python's array
wraparound bug (E722/E723/E724 errors -- circular dependencies, missing ROOT,
multiple ROOTs). Header documents the root cause and solution.

**Purpose:** Operational checklist for the corpus remediation campaign. The header
explained which error codes indicate broken `%gra` tiers.

**Still valid?** Same as above -- repeatable workflow, spent list.

#### `docs/wor-broken-files.txt` (2,863 paths)

**What it was:** Subset of %wor-affected files specifically from the Python
`decode()` bug (leaked nonwords into `%wor` tiers).

**Purpose:** More targeted remediation list (decode-bug files only).

**Still valid?** No. Subset of `FILES_NEED_ALIGN.txt`.

#### `docs/wor-rerun-dirs.txt` (390 directories)

**What it was:** Unique parent directories for all %wor-broken files. Used to
batch re-alignment at the directory level.

**Purpose:** Operational convenience -- `batchalign align` operates on directories,
so this converted the file list into directory-level commands.

**Still valid?** No. Derivable from any file list.

#### `docs/wor-rerun-list.txt` (437 files + 10-line header)

**What it was:** Curated rerun list with header documenting generation date
(2026-02-12), total counts (3,735 files, 390 dirs, 12 collections), root cause,
and submission instructions.

**Purpose:** Ready-to-submit remediation batch for the server.

**Still valid?** No. One-time operational artifact.

#### `throughput_report.txt` (181 lines)

**What it was:** Executive summary of throughput comparison between an early "ray"
distributed architecture and Python master. Discussed memory usage (6 GB vs 54 GB
for align), GPU bottlenecks, and cluster scaling.

**Purpose:** Performance justification document for an earlier architecture iteration.

**Still valid?** No. References the "ray" architecture which is no longer used.
Superseded by `~/batchalign-benchmarking/` results and `rust-migration-benchmark-report.md`.

**Book action for A2:** The *workflow pattern* (corpus-wide validation scan
producing per-collection file lists for batch remediation) is worth documenting.
See "Recurring Workflows" section at the end of this document.

## Part C: Recurring Workflows Worth Documenting

Several deleted/private files represent *repeatable workflows* that should be captured
in the book even though the specific outputs are disposable.

### C1. Corpus-Wide Validation Audit

**What:** Scan all ~99K CHAT files for specific error codes, cross-reference error
sources, produce per-collection file lists for batch remediation.

**Seen in:** `results/wor_errors/`, `FILES_NEED_ALIGN.txt`, `FILES_NEED_MORPHOTAG.txt`,
`CORPUS_AUDIT_REPORT.md`.

**How to reproduce:**
```bash
# Validate all files in a collection, output errors as JSONL
batchalign validate ~/data/childes-data/ --output corpus-audit.jsonl

# Filter for specific error codes
jq 'select(.errors[] | .code == "E722" or .code == "E724")' corpus-audit.jsonl

# Extract file paths for remediation
jq -r '.file' corpus-audit.jsonl | sort -u > files_needing_fix.txt
```

**Book location:** Developer Guide > Testing, or a new "Operations > Corpus Audits" section.

### C2. Cross-Branch Correctness Comparison

**What:** Run both branches on the same test data, diff the outputs tier-by-tier
to identify behavioral differences.

**Seen in:** `docs/bench_correctness_diffs/`, `CORRECTNESS_FINDINGS.md`,
`python-rust-discrepancies.md`.

**How to reproduce:** Use `~/batchalign-benchmarking/scripts/cross_branch_benchmark.py`
with `--correctness` flag.

**Book location:** Developer Guide > Testing, or Benchmarking section.

### C3. Worker Ceiling Discovery

**What:** Incrementally increase worker count until throughput degrades, to find
the optimal concurrency for a given machine and command.

**Seen in:** `BENCHMARK_METHODOLOGY.md`, `BENCHMARK_RESULTS.md`,
`morphotag-throughput-benchmarks.md`.

**How to reproduce:** Use `~/batchalign-benchmarking/scripts/run_benchmarks.py
--workers-sweep 1,2,4,8,12,16`.

**Book location:** Developer Guide > Benchmarking, or Operations > Performance Tuning.
